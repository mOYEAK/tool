## 1. 项目概述与基础架构

### 1.1 项目目标
开发一款轻量级、无需安装、即开即用的微信小程序工具箱。以个人开发者身份起步，前期通过**纯前端技术**实现功能，确保 **100% 快速通过微信官方审核**，后期逐步通过版本迭代演进。

### 1.2 主体状态栏（TabBar）配置
初版仅保留两个核心页面，结构极其精简：
1. **工具箱 (pages/index/index)**：采用网格（Grid）布局展示所有可用工具，分类清晰。
2. **我的 (pages/mine/mine)**：展示个人信息、流量主广告位置、联系客服、关于我们等。

### 1.3 技术栈选择
* **开发模式**：微信原生小程序开发（不使用 Uni-app 或 Taro，保证 AI 生成的原生 API 代码最稳健、体积最小）。
* **UI 组件库**：引入 **Vant Weapp** 提升视觉质感，减少纯写 CSS 的时间。
* **数据存储**：完全本地化，使用 `wx.setStorageSync` 存储用户本地配置。

## 2. 第一阶段：社交与实用工具（纯前端·高频·0风险）

*此阶段功能 100% 运行在手机本地，不依赖任何服务器，极易过审，作为小程序的“安全地基”。*

### 2.1 九宫格切图 / 长图拼接
* **功能描述**：
    * **九宫格切图**：用户上传一张图片，利用 Canvas 自动将其等分成 3x3 的九张图，并支持一键批量保存到相册。
    * **长图拼接**：用户选择多张图片，可调节排序，Canvas 自动计算总高度，垂直拼接为一张长图导出。
* **AI 开发核心逻辑**：
    * 调用 `wx.chooseMedia` 获取图片流。
    * 利用 `wx.createOffscreenCanvas`（离屏画布）或传统 Canvas 执行像素裁剪（`drawImage` 参数控制）。
    * 使用 `wx.saveImageToPhotosAlbum` 循环导出图片。

### 2.2 万能二维码生成与解码
* **功能描述**：
    * **二维码生成**：输入任意文本或网址，实时生成一张清晰的二维码，支持自定义中间的 Logo，支持保存到相册。
    * **二维码解码**：上传一张二维码图片，本地解析并提取其中的文本内容，支持一键复制。
* **AI 开发核心逻辑**：
    * **生成**：集成纯 JS 库 `weapp-qrcode.js`，通过 Canvas 渲染矩阵。
    * **解码**：直接调用微信原生 API `wx.scanCode({ sourceType: ['album'] })`，无需任何第三方图像识别库。

### 2.3 手写模拟器 / 签名生成
* **功能描述**：
    * 提供一个全屏/横屏的画板，用户可以用手指在屏幕上书写，支持调整画笔粗细、颜色。
    * 提供“一键清除”和“导出透明签名”功能。
* **AI 开发核心逻辑**：
    * 使用小程序 `<canvas>` 组件，绑定 `touchstart`、`touchmove`、`touchend` 事件。
    * 记录手指滑动的坐标轨迹，通过 `canvasContext.lineTo` 和 `stroke()` 实时绘制。
    * 导出时设置画布背景为透明，导出为 PNG 格式。

## 3. 第二阶段：办公与趣味设计（纯前端·进阶·低风险）

*此阶段重点解决用户在日常办公、个性化社交中的进阶需求，同样采用纯前端技术，但技术复杂度略有提升。*

### 3.1 纯前端图片压缩 / 格式转换
* **功能描述**：
    * **图片压缩**：用户上传大图，可通过滑块（Slider）自由微调目标清晰度（0.1~1.0），实时显示压缩后的预计文件大小。
    * **格式转换**：支持将 PNG、BMP、WEBP 等格式，在本地一键转换为微信最通用的 JPG/PNG 格式。
* **AI 开发核心逻辑**：
    * 利用Canvas的 `wx.canvasToTempFilePath` 接口。
    * 关键控制参数：`fileType` (指定 'jpg' 或 'png')，`quality` (控制压缩质量，0-1之间)。
    * 利用本地离屏画布规避页面渲染性能瓶颈。

### 3.2 精美带壳截图 / 壁纸制作
* **功能描述**：
    * **带壳截图**：用户上传一张手机截图，选择一种手机壳模板（如 iOS 扁平风、经典全面屏边框），系统自动把截图套入外壳中，生成高端的套壳分享图。
    * **壁纸制作**：支持纯色、渐变色背景选择，或在壁纸上添加一句本地文字（如励志语录），一键生成手机壁纸。
* **AI 开发核心逻辑**：
    * 准备几张透明背景的手机壳 PNG 边框素材（作为 Base64 内嵌在代码中，或者放在免流量的云存储中）。
    * 使用 Canvas 按照图层顺序绘制：底层绘制背景 -> 中层绘制用户截图 -> 顶层绘制手机壳模板图片。

## 4. 第三阶段：音视频与去水印（需后端·高流·高风险）

*核心警告：由于微信官方对个人开发者“音视频服务”和“版权侵权”的严格限制，此类功能不可在第一版提交审核！必须等前两阶段积累好基础用户，且你有条件升级为“企业主体”时，再行推出。*

### 4.1 短视频提取 & 视频转音频
* **功能描述**：
    * 用户复制抖音、快手、小红书等平台的分享链接，输入到小程序中，一键解析出“无水印短视频”下载地址。
    * 支持“视频转音频”功能，提取视频中的背景音乐并保存为 MP3。
* **架构设计（必须接入后端）**：
    * **前端（小程序）**：仅负责接收链接文本，并使用 `wx.downloadFile` 下载后端处理好的媒体流。
    * **后端（Python/Node.js）**：部署在云服务器。利用爬虫解析第三方平台的真实播放地址；利用 `FFmpeg` 命令行工具，将视频流中的音频轨道切出并转码为 MP3（命令如：`ffmpeg -i input.mp4 -vn -acodec libmp3lame output.mp3`）。

### 4.2 图片去水印
* **功能描述**：
    * 用户上传带有水印的图片，用手指涂抹水印区域，系统自动通过算法“擦除”水印，恢复背景。
* **架构设计**：
    * 前端使用 Canvas 捕捉用户的“涂抹蒙版（Mask）”。
    * 将原图与蒙版上传到后端服务器，调用图像修复算法（如 OpenCV 传统的 `inpaint` 方法，或者开源的 AI 修复模型如 Lama-Cleaner）处理后，返回给前端。

## 5. 全局 UI 与交互规范

为确保 AI 生成的代码具备一致的视觉体验，开发时请严格遵循以下规则：

1.  **色彩主调**：采用去饱和度的专业科技感配色（如：深灰色 `#323233` 作为主文字，淡雅灰 `#F7F8FA` 作为全局背景，清爽蓝 `#1989FA` 作为高亮激活态）。
2.  **首页布局**：
    * 顶部：一个简洁的 Banner 或搜索框（“搜索你想要的工具”）。
    * 中部：采用 Vant Weapp 的 `van-grid` 组件，四列或三列网格展示，图标配合文字。
    * 按功能分类目：如【图像处理】组、【日常实用】组。
3.  **异常处理**：
    * 涉及保存到相册的功能，AI 必须生成“检查保存权限”的代码（`wx.getSetting`），如果用户拒绝，提示引导其去设置页开启。
    * 所有长耗时操作（Canvas 渲染、文件下载），必须包裹 `wx.showLoading({ title: '处理中...' })`，完成后执行 `wx.hideLoading()`。

## 6. 给 AI 辅助编程 (Vibe Coding) 的专属提示词指令

当你准备让 Codex/Cursor 帮你写代码时，请分批、精准地输入以下 Prompt：

### 🧱 步骤一：初始化项目底座（在项目根目录执行）
> “我正在开发一个微信原生小程序，请帮我配置全局的 `app.json`。要求：
> 1. 包含底部的 TabBar，只有两个标签：一个是‘工具箱’（路径 `pages/index/index`），另一个是‘我的’（路径 `pages/mine/mine`）。
> 2. 全局窗口背景色设为 `#F7F8FA`，导航栏背景色设为 `#FFFFFF`，文字颜色设为黑色。
> 3. 请同时给出 `pages/index/index` 和 `pages/mine/mine` 的基础 .wxml, .js, .wxss 页面初始化代码。”

### 🎨 步骤二：让 AI 帮你写九宫格切图（进入 index 页面后提问）
> “请在工具箱中实现‘九宫格切图’功能。请使用微信原生 WXML、WXSS 和 JS 实现：
> 1. 提供一个按钮让用户上传手机图片。
> 2. 图片上传后，使用离屏 Canvas（或原生 Canvas）将图片精准裁剪成 3x3 的 9 张等大正方形图片，并在前端页面用网格整齐预览这 9 张图。
> 3. 提供一个‘一键保存全部’按钮。点击后，通过循环调用微信 API，将这 9 张图全部安全保存到用户的系统相册中，并处理好用户可能拒绝相册权限的异常。请确保不使用任何浏览器 DOM API。”

### 🚀 步骤三：依此类推
每当你需要开发 PRD 中的下一个功能时，直接复制 PRD 中该功能对应的【功能描述】与【AI 开发核心逻辑】丢给 AI 即可。这种精细化的“切片提问”能保证 Vibe Coding 的代码准确率达到 95% 以上。
