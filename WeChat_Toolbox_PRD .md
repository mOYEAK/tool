# 独立开发者「万能工具箱」微信小程序 PRD 修订版

## 1. 项目概述与基础架构

开发一款微信原生小程序工具箱，前期用纯前端高频工具积累用户，后期逐步加入轻后端、AI、广告解锁和按次付费能力。

基础结构保持不变：

- `pages/index/index`：工具箱首页，按分类展示工具
- `pages/mine/mine`：我的页面，展示使用记录、流量主广告位、客服、关于我们、隐私协议
- 技术栈：微信原生小程序 + Vant Weapp + `wx.setStorageSync`
- 初期不强依赖后端，支付和广告能力先预留配置

核心原则：

- 每个工具独立模块化，方便以后单独拆出
- 前三阶段优先低风险、低资质依赖
- 第四阶段保留高流量高风险功能，但默认最后做
- 收费方式按功能配置，不做全局强绑定

## 2. 第一阶段：纯前端免费工具

第一阶段负责快速上线、积累 UV 和验证用户需求，全部功能默认免费。

- 九宫格切图：上传图片，本地 Canvas 裁成 9 张图，支持批量保存
- 长图拼接：选择多张图片，调整顺序后拼接成长图
- 二维码生成与解码：输入文本/网址生成二维码；从相册扫码并复制内容
- 手写签名：Canvas 手写，支持笔色、粗细、清除、透明 PNG 导出
- 图片压缩/格式转换：本地压缩图片，支持 JPG/PNG 转换和质量滑块
- 带壳截图/壁纸制作：截图套壳、纯色/渐变壁纸、文字壁纸生成

## 3. 第二阶段：办公与活动工具

第二阶段加入更实用的办公工具。初期免费限次，后续逐步支持看广告解锁或按次导出。

- PDF 转换 / PDF 工具：
  - 图片转 PDF：优先纯前端实现，适合先上线
  - PDF 转图片：可先做简单版，复杂能力后端化
  - PDF 合并/拆分：第二阶段后半段实现
  - PDF 压缩：简单压缩先做，复杂压缩后端化
  - PDF 转 Word/Excel：放到后续增强版，通常需要后端或 OCR

- 图片转表格 / OCR 提取：
  - 用户上传图片
  - 系统识别表格或文本
  - 免费预览结果，后续完整 Excel 导出可付费或看广告

- 报价单 / 收据模板：
  - 用户填写店名、项目、金额、备注等字段
  - 生成图片或 PDF
  - 适合个体户、小商家、自由职业者使用

- 活动海报生成：
  - 用户上传图片、输入活动标题和时间地点
  - 套用本地模板生成海报
  - 普通版免费，高清无水印版后续可看广告或付费

- 活动报名 / 随机抽取工具：
  - 首页展示名建议用「活动报名」「报名接龙」「名单抽取」，不要直接主打“抽奖”
  - 本地抽取版：用户手动输入名单，本地随机抽取，不需要后端
  - 报名版：创建报名表，收集姓名、手机号、备注等信息，需要后端和隐私说明
  - 禁止做付费抽奖、现金抽奖、充值抽奖、诱导分享抽奖、拉人头抽奖
  - 后续变现方式：高级模板、导出 Excel、人数上限扩容、去水印海报

## 4. 第三阶段：AI 与商业化工具

第三阶段开始加入更强的付费点，但仍保持“用户上传自己的素材，系统返回本人使用结果”的工具定位。

- 证件照生成：
  - 普通图免费生成
  - 高清图、美颜图、换底色高清导出后续付费
  - 当前未满流量主/支付条件时，先保留高清能力入口占位

- 图片转提示词：
  - 用户上传自己的图片
  - 系统分析风格并生成专属提示词
  - 不做公共提示词库，不做付费解锁内容库

- 简历优化：
  - 基础建议免费
  - 完整 PDF、多版本优化、高清导出后续付费

- 朋友圈文案 / 短视频标题：
  - 用户输入主题、产品或场景
  - 系统生成个人使用文案
  - 短结果免费，完整结果可看广告或付费

## 5. 第四阶段：高流量后再做的高风险功能

第四阶段功能保留在规划中，但不进入首发版本，不默认展示入口。

- 短视频无水印：
  - 用户输入第三方平台分享链接
  - 后端解析并返回视频
  - 必须加入版权提示，仅允许处理本人有权使用的素材

- 视频转音频：
  - 用户上传视频或提供可处理视频
  - 后端使用 FFmpeg 提取音频
  - 默认按次付费或限制次数

- 图片去水印：
  - 用户上传图片并涂抹区域
  - 后端修复后返回图片
  - 页面必须提示仅处理本人拥有权利或授权的图片

第四阶段上线条件：

- 前三阶段已有稳定 UV 和留存
- 后端、内容安全、隐私协议、版权提示准备完整
- 入口支持后台开关和灰度
- 审核版本可关闭高风险入口
- 不与第一、第二阶段工具强耦合

## 6. 收费与广告策略

整体采用「免费基础功能 + 广告解锁 + 按次付费导出」模式。

- 第一阶段：全部免费，流量主开通后可加 Banner 或插屏
- 第二阶段：免费限次，后续完整导出、高清导出、无水印导出可看广告或付费
- 第三阶段：证件照、OCR、简历优化、AI 文案、图片转提示词逐步商业化
- 第四阶段：默认按次付费，不建议无限免费

支付未准备好之前：

- 保留支付配置和商品配置
- 真实支付能力先只预留，不强依赖
- 证件照先做普通图免费，高清美颜图显示“即将开放”
- 流量主未开通前，激励视频入口显示“观看广告解锁暂未开放”

## 7. 模块化技术设计

建议按工具注册表驱动首页和功能入口。

工具配置字段：

- `key`：工具唯一标识
- `name`：展示名称
- `category`：所属分类
- `phase`：阶段
- `pagePath`：页面路径
- `monetization`：`free | quota | rewardAd | pay | payOrAd`
- `requiresBackend`：是否需要后端
- `enabled`：是否展示入口

建议目录：

- `pages/tools/{toolKey}`：每个工具一个独立页面
- `modules/tools`：工具注册表
- `services/canvas`：图片处理、裁剪、导出
- `services/quota`：免费次数、每日次数、本地记录
- `services/ad`：流量主广告适配
- `services/payment`：微信支付适配，前期占位
- `services/backend`：后端 API 统一封装

验收标准：

- 首页分类、搜索、跳转正常
- 每个纯前端工具断网可用
- 相册保存权限流程完整
- 免费次数、广告解锁、支付占位互不影响
- 未开通流量主或支付时，用户不会卡死
- 第四阶段入口默认关闭
- 每个工具可以独立复制出去，最少依赖全局代码
