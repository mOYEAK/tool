# 独立开发者「万能工具箱」微信小程序 PRD 修订版

## 1. 项目概述与基础架构

保留原框架：微信原生小程序 + Vant Weapp + 本地存储 + 两个 TabBar 页面。

产品定位调整为：**低门槛、高频、可逐步商业化的图片/文件/办公工具箱**。前期以纯前端免费工具积累 UV，后期逐步加入轻后端、AI、广告解锁和按次付费能力。

TabBar 保持不变：

- `pages/index/index`：工具箱首页，按分类展示工具
- `pages/mine/mine`：我的页面，展示使用记录、广告位、客服、关于我们、隐私协议

核心原则：

- 前三阶段尽量低风险、低资质依赖
- 第四阶段保留短视频无水印、视频转音频、图片去水印，但必须最后做
- 每个工具独立成模块，方便以后单独拆出一个小程序
- 付费、广告、限次逻辑按工具配置，不写死在业务页面里

## 2. 分阶段功能规划

第一阶段：纯前端免费工具，负责拉新和过审基础

- 九宫格切图：上传图片，本地 Canvas 裁成 9 张图，支持批量保存
- 长图拼接：多图排序后本地拼接成长图
- 二维码生成与解码：文本/网址生成二维码，扫码相册二维码并复制结果
- 手写签名：本地 Canvas 书写，导出透明 PNG
- 图片压缩/格式转换：本地压缩 JPG/PNG，支持质量滑块
- 带壳截图/壁纸制作：本地套模板、纯色/渐变壁纸生成

第二阶段：低风险进阶工具，先免费限次，后续按次导出

- PDF 压缩：前期可先做图片转 PDF/简单压缩，复杂 PDF 压缩后端化
- 图片转表格/OCR 提取：用户上传图片，返回表格或文本结果
- 报价单/收据模板：本地填写字段，生成图片或 PDF
- 活动海报生成：本地模板 + 用户文字/图片，导出海报
- 简历优化普通版：用户输入文本，生成优化建议或排版稿

第三阶段：AI 与商业化工具，少量核心功能开始收费或看广告

- 证件照生成：免费生成普通预览图；高清、美颜、换底色版本走付费或广告解锁
- 图片转提示词：用户上传自己的图，生成专属提示词，不做公共提示词库
- 朋友圈文案/短视频标题：用户输入主题，生成个人使用结果
- 简历优化高级版：生成完整 PDF、高清导出、多个版本对比
- 图片转 Excel 高级版：免费预览识别结果，高清/完整 Excel 导出付费

第四阶段：高流量后再做的高风险功能，默认不首发、不默认展示

- 短视频无水印：用户输入链接，后端解析，返回可下载视频
- 视频转音频：用户上传或提供视频，后端 FFmpeg 提取音频
- 图片去水印：用户上传图片并涂抹区域，后端修复后返回图片

第四阶段上线条件：

- 前三阶段已有稳定 UV 和留存
- 小程序主体、类目、隐私协议、内容安全和版权提示已准备好
- 后端独立部署，和主工具箱业务解耦
- 功能入口可灰度开关，审核版本可关闭
- 页面明确提示用户仅处理本人拥有权利或授权的素材

## 3. 收费与广告策略

采用「免费基础能力 + 后续功能差异化变现」模式。

功能级变现配置：

- 第一阶段工具：全部免费，不接支付；流量主开通后可放 Banner/插屏，但不影响主流程
- 第二阶段工具：免费限次，后续可按次导出；未接支付前只做次数记录和导出限制预留
- 证件照：普通图免费，高清图/美颜图/换底色高清导出付费
- 图片转 Excel：免费预览前几行或低额度识别，完整导出付费或看激励视频
- 活动海报/壁纸：普通水印版免费，无水印高清版看广告或付费
- 简历优化：基础建议免费，完整排版 PDF 或多版本优化付费
- 图片转提示词：短结果免费，完整参数/多平台版本看广告或付费
- 第四阶段功能：默认按次付费，不建议免费无限使用

支付未准备好之前的默认策略：

- 保留商品配置和支付入口占位
- 证件照先实现“普通图免费，高清美颜图暂未开放/后续开放”
- 流量主未达门槛前，激励视频入口使用普通提示替代
- 流量主开通门槛以微信公众平台后台实际提示为准，你现在可先按 500 UV 目标规划

## 4. 模块化与技术设计

保持微信原生小程序，不引入 Uni-app/Taro。

建议目录结构：

- `pages/index`：工具箱首页，只负责展示工具入口
- `pages/mine`：我的页面
- `pages/tools/{toolKey}`：每个工具一个独立页面目录
- `modules/tools`：工具注册表，配置名称、图标、分类、阶段、是否收费、是否需后端
- `services/ad`：激励视频/Banner/插屏广告适配层
- `services/payment`：微信支付适配层，前期只保留接口占位
- `services/quota`：免费次数、每日次数、本地记录
- `services/canvas`：图片绘制、裁剪、导出公共能力
- `services/backend`：后端 API 请求封装，第四阶段单独接入

工具注册表必须支持：

- `key`：工具唯一标识
- `name`：展示名称
- `category`：分类
- `phase`：阶段
- `pagePath`：页面路径
- `monetization`：`free | quota | rewardAd | pay | payOrAd`
- `requiresBackend`：是否需要后端
- `enabled`：是否展示入口

这样后续想把「证件照」或「图片转表格」单独拆成一个小程序，只需要复制对应页面、服务和配置。

## 5. UI、审核与验收标准

UI 保留原规范：

- 全局背景 `#F7F8FA`
- 主文字 `#323233`
- 高亮色 `#1989FA`
- 首页顶部搜索框 + 分类 Grid
- 每个工具页结构统一：上传区、参数区、预览区、导出按钮

所有工具必须具备：

- 上传前说明用途
- 处理时 `wx.showLoading`
- 失败时明确提示
- 保存相册权限检查与引导
- 用户文件仅用于当前处理，不做公开展示
- 后端功能必须有隐私说明和删除策略

验收测试：

- 首页工具分类、搜索、跳转正常
- 每个纯前端工具断网可用
- 相册保存授权、拒绝、再次授权流程可用
- 免费次数、广告解锁、付费占位逻辑互不影响
- 未开通流量主时不会阻塞功能
- 未开通支付时不会出现真实支付失败死路
- 第四阶段入口默认关闭，打开后也不影响前面工具审核和使用

参考依据：

- 微信小程序广告能力与激励广告接口以微信官方开发文档为准：[wx.createRewardedVideoAd](https://developers.weixin.qq.com/miniprogram/dev/api/ad/wx.createRewardedVideoAd.html)
- 小程序备案与类目以微信公众平台后台和官方备案文档为准：[小程序备案](https://developers.weixin.qq.com/miniprogram/product/record/)
- 流量主门槛和广告位类型以微信公众平台后台实际提示为准。
